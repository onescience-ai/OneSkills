# Custom Skill Contribution Guide

本指南面向 `OneSkills` 用户与贡献者。

目标是帮助你在不破坏 `OneSkills` 产品分层的前提下，添加自己的领域经验或自定义技能。

## 一、先判断你要补什么

在新增内容前，先判断它属于哪类：

- 领域理解
- 角色协作
- 模板资产
- 新的稳定执行能力

推荐顺序是：

1. 先补领域理解
2. 再补角色协作
3. 再补模板资产
4. 只有必要时才新增 skill

## 二、优先补哪里

### 1. 领域理解

如果你想补：

- 某个科研领域的任务画像
- 用户在该领域里的常见表达方式
- 常见数据对象和约束

优先补到：

- `skills/onescience-workflow/references/`

### 2. 角色协作

如果你想补：

- 这个领域里谁先做、谁后做
- 角色之间如何交接

优先补到：

- `skills/onescience-role/references/`

### 3. 模板资产

如果你想补：

- 数据读取模板
- 配置模板
- 模型改造模板
- 训练 / 推理脚本模板

优先补到：

- `skills/onescience-coder/assets/`
- `skills/onescience-coder/references/`

## 三、什么时候才建议新增 skill

只有在下面条件同时满足时，才建议新增：

- 现有 skill 无法承接
- 新能力会被反复复用
- 它有独立输入、输出和边界
- 它不是对现有执行技能的领域化复制

## 四、不建议的做法

- 按领域复制一整套 `coder/debug/runtime`
- 只换个名字就新增一个平行 skill
- 把单项目临时经验直接写成通用技能
- 把某个智能体的使用偏好误当成通用规则

## 五、推荐提交流程

1. 先写清问题与目标
2. 判断所属层级
3. 做最小改动
4. 补必要文档
5. 再提交 PR

## 六、一句话原则

优先扩展现有分层，尽量不要复制 skill；只有新增稳定执行能力时，才新增新的 skill。
