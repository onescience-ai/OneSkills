# 数据质量检查清单

本文档定义数据集验证阶段的质量检查项和执行方法。

**注意**：以下示例代码仅供参考，实际实现应根据数据格式和需求灵活调整。

## 检查分类

### 1. 完整性检查 (Completeness)

**目的**：验证数据集是否包含所有必要的组件。

**检查项**：
- train/val/test 目录是否存在
- 每个 split 是否包含数据文件
- 样本数是否符合预期（非空）
- 必要的字段/变量是否存在

**检查逻辑**：
- 验证 train/val/test 目录是否存在
- 检查每个 split 是否包含数据文件
- 验证样本数是否符合最低要求

### 2. 格式验证 (Format)

**目的**：验证数据格式是否正确。

**检查项**：
- 文件扩展名是否正确
- 文件是否可以正常读取
- 数据维度是否符合预期
- 数据类型是否正确

**检查逻辑**：
- 验证文件扩展名是否与预期格式匹配
- 尝试读取文件头部，确认文件可以正常打开
- 检查数据维度和类型是否符合预期

### 3. 统计分布检查 (Statistics)

**目的**：验证数据的统计特性是否合理。

**检查项**：
- 缺失值比例
- 异常值数量
- 数值范围
- 标签分布
- 特征相关性

**检查逻辑**：
- 计算缺失值比例，与阈值对比
- 使用统计方法（如IQR）检测异常值
- 分析标签分布是否平衡
- 检查数值范围是否合理

### 4. 领域特定检查 (Domain-Specific)

**目的**：根据数据领域执行专门的检查。

#### 地球科学数据

**检查项**：
- 时间维度连续性
- 空间范围完整性（lat/lon覆盖）
- 变量单位正确性
- 物理量合理性（温度、压力等）

#### 生物信息数据

**检查项**：
- 批次效应检测
- 基因表达值范围合理性
- 样本质控指标
- 归一化状态

#### 材料化学数据

**检查项**：
- 能量值合理范围
- 结构完整性
- 原子坐标有效性
- 组成元素合理性

## 检查优先级

1. **必须检查**：完整性、格式验证
2. **推荐检查**：统计分布、领域特定检查
3. **可选检查**：数据质量的深度分析

## 检查失败策略

- **严重错误**（完整性、格式）→ `status: failed`，阻止后续流程
- **统计异常**（分布、异常值）→ `status: warning`，记录但不阻止
- **领域特定问题** → 根据严重程度决定

## 注意事项

- 检查应该高效，避免加载全部数据（可抽样）
- 对于大数据集，使用流式处理或分块读取
- 检查结果应包含具体的数值和阈值，便于调试
- 为每个检查提供修复建议
