# 知识获取策略

## 强制规则（先读）

知识分两类，来源不可混用：

- **OneScience 具体资源**（特定模型、datapipe、工具、原语的数据处理方法）：**必须调用 type=resource 技能获取**。**严禁**通过搜索、翻阅项目目录或本技能内文档来替代资源召回。
- **领域判断 / 平台元信息 / 通用处理模式**：可使用本技能的 `./assets/` 与 `./references/`，仅用于辅助决策与通用动作编排，不能替代上述资源召回。

下方"知识来源优先级"只适用于第二类（领域/平台/通用知识）；第一类资源召回不受其约束，始终走 type=resource 技能。

## 知识来源优先级

### 1. 平台数据资源信息（最高优先级）

当用户使用 OneScience 平台默认数据时：

- **ERA5 数据**：读取 `./assets/era5_metadata.json` 了解平台 ERA5 数据的变量、时空范围、分辨率等信息
- 其他平台数据：类似方式读取对应的元信息文件

### 2. 本技能沉淀知识

- 使用 `./references/data_processing_mapping.md` 中的通用数据处理映射规则
- 使用本技能历史积累的处理模式和最佳实践

### 3. 资源技能召回

当需要特定模型或任务的数据处理知识时（**此为强制路径，不是可选兜底**）：

- 先消费 `onescience-orchestrator` 传入的 `available_resource_summaries`
- 内容不足时，**必须调用 type=resource 技能**获取特定模型的数据处理方法
- 不得用搜索本地/项目文档的方式替代此步

### 4. 在线文献检索（最后手段）

**触发条件**：

- 用户明确指定了特定模型（如 pangu-weather、fourcastnet 等）
- 前述三种来源均无法提供充分的数据处理信息

**检索策略**：

- 搜索模型相关的论文、官方文档、GitHub 仓库
- 提取数据处理相关的关键信息：
  - 输入数据格式和变量
  - 预处理步骤
  - 数据分辨率和时空范围
  - 标准化方法
  - 数据划分方式

**注意事项**：

- 仅提取数据处理相关内容，不获取模型结构、训练方法等
- 验证信息的权威性和时效性
- 在规划中标注知识来源

## 知识整合原则

- 平台数据信息是事实，直接使用
- 通用处理知识是基础模式，可灵活应用
- 特定模型知识是参考，需结合实际情况调整
- 在线检索知识需标注来源和可信度
