# 数据处理需求映射

## 映射原则

将用户的数据处理需求映射为具体的处理动作序列。

## 常见需求到处理动作的映射

### 数据集构建类

| 需求描述 | 处理动作 | 关键参数 |
|---------|---------|---------|
| 构建模型训练数据集 | 数据获取 → 格式转换 → 特征对齐 → 标准化 → 划分（train/val/test） | 分辨率、时间范围、变量列表、划分比例 |
| 构建时序预测数据集 | 时间对齐 → 窗口化 → 序列构建 → 归一化 | 窗口大小、步长、预测长度 |
| 构建空间数据集 | 空间对齐 → 重网格化 → 区域裁剪 → 投影转换 | 目标网格、投影类型、裁剪范围 |

### 数据预处理类

| 需求描述 | 处理动作 | 关键参数 |
|---------|---------|---------|
| 数据清洗 | 缺失值处理 → 异常值检测 → 质量标记 | 缺失阈值、异常值标准 |
| 数据对齐 | 时间对齐 → 空间对齐 → 插值 | 目标时间分辨率、空间分辨率、插值方法 |
| 数据标准化 | 单位转换 → 归一化/标准化 | 目标单位、标准化方法 |

### 数据转换类

| 需求描述 | 处理动作 | 关键参数 |
|---------|---------|---------|
| 格式转换 | 读取源格式 → 转换为目标格式 | 源格式、目标格式 |
| 空间重采样 | 重网格化 → 插值 | 目标分辨率、插值方法 |
| 时间重采样 | 时间聚合/插值 | 目标时间分辨率、聚合方法 |

## 知识获取优先级

> 注意：涉及具体模型 / datapipe / 工具的数据处理知识，**必须调用 type=resource 技能获取**，不得用搜索本地/项目文档替代。下列优先级仅适用于通用处理模式与平台元信息这类辅助知识。

1. **本地沉淀知识**：使用 references 中的通用处理映射规则（仅通用动作编排）
2. **平台资源信息**：读取 assets 中的平台数据元信息（如 era5_metadata.json）
3. **资源技能召回（具体模型/datapipe 知识的强制来源）**：调用 type=resource 技能获取特定模型/任务的数据处理知识，不得用本地文档替代
4. **在线检索（最后手段）**：如有明确的模型名称且 type=resource 召回不足，可联网检索相关文献获取数据处理方法，并标注来源

## 输出规划结构

每个处理动作应包含：

- **动作名称**：清洗、筛选、转换、对齐、重采样、聚合等
- **输入要求**：数据格式、必需字段、质量要求
- **输出产物**：数据格式、字段说明、元信息
- **依赖关系**：前置步骤
- **风险点**：数据丢失、精度损失、时间消耗等
