# 气象领域通用数据处理知识

## 一、气象数据契约

规划气象数据处理前，先明确数据契约，避免把变量、时间、空间或物理含义处理错。

| 契约项 | 需要确认的内容 | 处理影响 |
|---|---|---|
| 数据来源 | 观测、再分析、模式预报、卫星反演、雷达、站点、集合预报 | 决定坐标体系、误差结构、时效性和可比性 |
| 变量名称与标准名 | short name、long name、CF standard_name、平台变量名 | 变量映射、单位转换、派生量计算 |
| 维度结构 | time、level、latitude、longitude、station、member、lead_time、valid_time | 对齐、切片、拼接、样本构建 |
| 时间含义 | 分析时刻、预报起报时刻、预报时效、有效时刻、累计窗口 | 时序对齐、标签构建、累计量差分 |
| 空间坐标 | 经纬度网格、投影网格、站点坐标、扫描坐标 | 重网格、插值、区域裁剪 |
| 垂直坐标 | pressure level、model level、height、depth、surface | 层选择、垂直插值、剖面构建 |
| 单位与符号约定 | K/degC、Pa/hPa、m/s、kg m-2 s-1、m、geopotential/geopotential height | 数值可比性和物理公式正确性 |
| 缺测与掩膜 | NaN、_FillValue、missing_value、海陆掩膜、地形掩膜 | 有效样本筛选、统计分母、模型掩膜 |
| 分辨率 | 时间步长、空间网格距、垂直层间距 | 重采样策略、别名风险、计算成本 |
| 数据格式 | NetCDF、GRIB、Zarr、HDF、GeoTIFF、CSV/Parquet | 读取方式、分块策略、元数据保留 |

## 二、变量类型与处理要点

| 变量类型 | 常见变量 | 处理要点 |
|---|---|---|
| 状态量 | 温度、气压、湿度、位势高度、海温 | 可做均值、插值、距平；注意垂直层和单位 |
| 通量量 | 短波/长波辐射、感热、潜热、水汽通量 | 明确方向符号、时间平均或累计窗口 |
| 累计量 | 降水、蒸发、辐射累积、雪水当量变化 | 先确认累计起点和窗口，再差分或聚合 |
| 向量量 | u/v 风、洋流、通量矢量 | 分量需同层同网格；派生风速、风向时保留分量 |
| 分类/掩膜量 | 海陆掩膜、云类型、天气类型、土壤类型 | 避免连续插值；重网格时使用最近邻或类别占比 |
| 站点量 | 站点温度、降水、风、气压 | 需站点元数据、海拔、时区、观测制度 |
| 集合量 | member 维度、概率预报 | 保留成员维度或计算集合统计，避免过早平均 |

## 三、单位、物理量与派生变量

优先读取数据元信息中的单位；只有元信息明确时才进行转换。常见转换和派生关系如下。

| 处理对象 | 常见规则 |
|---|---|
| 温度 | K 转 degC：`T_C = T_K - 273.15`；模型输入可保留 K，但同一数据集内必须统一 |
| 气压 | Pa 与 hPa 转换：`hPa = Pa / 100`；压力层变量需明确层位单位 |
| 位势高度 | 若变量是 geopotential，转换为位势高度：`z = geopotential / g`，`g` 约为 9.80665 m s-2 |
| 风速 | `wind_speed = sqrt(u^2 + v^2)`，u/v 必须来自同一高度或压力层 |
| 风向 | 气象风向（度）：`atan2(-u, -v) * 180/π`，再调整到 0-360°；气象风向表示风从哪个方向来 |
| 降水 | 累计降水需按时间差分得到分时段降水：`precip[t] = cumulative[t] - cumulative[t-1]`；差分后负值通常需要标记为重置或异常 |
| 相对湿度/比湿 | 相对湿度转比湿需饱和水汽压：`e_s = 6.112 * exp(17.67*T_C/(T_C+243.5))`，`q = 0.622*RH*e_s/(p - RH*e_s)`，其中 T_C 为摄氏度，p 为 hPa |
| 距平 | `anomaly = value - climatology`；气候态需匹配日序、月份、季节或滑动窗口 |
| 标准化距平 | `(value - climatology_mean) / climatology_std`；需避免样本过少导致标准差不稳定 |
| 梯度 | 经纬度网格：`∂f/∂x ≈ (f[i+1,j] - f[i-1,j]) / (2*Δx*cos(lat))`，`∂f/∂y ≈ (f[i,j+1] - f[i,j-1]) / (2*Δy)`，Δx、Δy 为弧度距离 |
| 涡度 | `ζ = ∂v/∂x - ∂u/∂y`；经纬度网格需使用球面坐标公式或有限差分近似 |
| 散度 | `D = ∂u/∂x + ∂v/∂y`；经纬度网格需考虑纬度权重：`D ≈ (∂u/∂x)/cos(lat) + ∂(v*cos(lat))/∂y/(cos(lat)*R)` |

## 四、时间处理

气象数据的时间维度通常比普通时间序列更复杂。处理时先区分“数据代表的时间”。

| 时间对象 | 处理方法 |
|---|---|
| 观测时刻 | 统一时区，记录原始时区；站点日统计需明确本地日或 UTC 日 |
| 分析时刻 | 用作再分析或同化产品的有效时刻，通常可按 time 对齐 |
| 预报起报时刻 | 保留 initialization/reference time，不能直接与 valid_time 混淆 |
| 预报时效 | lead_time 与 reference time 共同确定 valid_time |
| 有效时刻 | 多源对齐、标签匹配和验证时优先使用 valid_time |
| 累计窗口 | 降水、辐射等累计量需明确从何时到何时累计 |

常用时间处理动作：

- 排序并去重时间坐标，检测重复时刻和不规则间隔。
- 将多源数据对齐到共同有效时刻，必要时保留时间容差。
- 对瞬时状态量可做重采样均值、最大、最小或插值；对累计量应先差分或按窗口聚合。
- 构建日、候、旬、月、季节、年统计时，明确日界线、季节定义和闰日策略。
- 训练样本构建时，使用滑动窗口、滞后特征、未来标签或多步标签时，避免未来信息泄漏。
- 长期趋势或气候态处理需使用足够长的时间样本，并记录基准期。

### 时间对齐与重采样详细方法

| 处理类型 | 适用场景 | 具体方法 |
|---|---|---|
| 时间对齐 | 多源数据合并到共同时刻 | 使用 `valid_time` 作为对齐键；设置容差（如 ±30min）匹配最近时刻 |
| 时区转换 | 站点本地时间转 UTC | `time_utc = time_local - timezone_offset`；记录原始时区信息 |
| 预报时刻计算 | 从起报时刻和时效计算有效时刻 | `valid_time = reference_time + lead_time`；注意时间单位统一 |
| 累计量差分 | 总降水转时段降水 | `precip[t] = cumulative[t] - cumulative[t-1]`；负值标记为异常或重置 |
| 重采样-瞬时量 | 小时数据聚合到日/月 | 均值：`mean(values)`；极值：`max/min(values)`；时刻：`resample(method='nearest')` |
| 重采样-累计量 | 小时累计聚合到日累计 | 求和：`sum(hourly_increments)`；不可直接平均累计值 |
| 滑动窗口 | 移动平均、滞后特征 | `rolling_mean[t] = mean(values[t-w:t])`；避免窗口包含未来数据 |
| 气候态计算 | 多年同期平均 | 按日序/月份分组：`climatology[doy] = mean(values[year, doy])`，需 ≥10 年 |
| 时间插值 | 填补缺失时刻 | 线性插值：`value(t) = v1 + (v2-v1)*(t-t1)/(t2-t1)`；仅适用瞬时量 |

## 五、空间处理

| 数据形态 | 常用处理方法 | 风险点 |
|---|---|---|
| 规则经纬度网格 | 区域裁剪、重网格、面积加权平均、空间插值 | 高纬格点面积差异，经度 0/360 与 -180/180 |
| 投影网格 | 投影转换、重投影、插值到目标网格 | 投影参数缺失会导致位置偏移 |
| 非结构网格 | 保留网格拓扑，使用专用插值或邻近搜索 | 不能假设二维经纬度矩阵规则排列 |
| 站点数据 | 站点筛选、空间匹配、邻近格点提取、客观分析 | 海拔差异和站点迁移会引入偏差 |
| 卫星/雷达扫描 | 质量标识过滤、投影或重采样、轨道拼接 | 观测角度、遮挡和扫描几何影响误差 |

空间处理原则：

- 经纬度裁剪需处理跨 180 度经线区域。
- 区域平均优先使用面积权重；规则经纬度网格常用 `cos(latitude)` 权重。
- 连续变量可使用双线性、保守或高阶插值；分类变量和掩膜使用最近邻或类别占比。
- 降水、辐射通量等守恒量重网格时优先考虑保守重映射。
- 多源融合前统一空间网格、垂直层和时间含义，避免只对齐文件形状。
- 地形敏感变量（近地面温度、地表气压、降水）跨分辨率处理时需记录地形差异风险。

### 重网格与插值详细方法

| 插值方法 | 适用变量 | 具体实现 |
|---|---|---|
| 最近邻 | 分类变量、掩膜、云类型、土地类型 | 找距离目标点最近的源格点：`value_target = value_source[argmin(distance)]` |
| 双线性插值 | 温度、气压、湿度等连续状态量 | 四个邻近格点加权：`f(x,y) = f00*(1-dx)*(1-dy) + f10*dx*(1-dy) + f01*(1-dx)*dy + f11*dx*dy` |
| 保守重映射 | 降水、辐射、通量等守恒量 | 计算源格点与目标格点重叠面积权重：`value_target = Σ(value_source * overlap_area) / target_area` |
| 距离反比权重 | 站点数据插值到网格 | `value(x,y) = Σ(value_i / d_i^p) / Σ(1/d_i^p)`，p 通常取 2，d 为距离 |
| 样条插值 | 高分辨率需求、平滑场 | 使用三次样条或双三次插值，适合光滑变量但计算成本高 |
| 面积加权平均 | 高分辨率降采样到低分辨率 | `value_coarse = Σ(value_fine * weight) / Σ(weight)`，weight 为格点面积，经纬度网格用 `cos(lat)` |
| 跨经度边界处理 | 全球数据跨 0°/360° 或 ±180° | 统一经度范围到 [0,360) 或 [-180,180)；裁剪时特殊处理跨界区域 |

### 卫星/雷达数据空间处理详细方法

| 处理步骤 | 具体方法 | 关键参数 |
|---|---|---|
| 质量标识过滤 | 读取 quality_flag 或 QC 字段，按位掩码或枚举值筛选；保留"good"或"acceptable"标识样本，剔除"bad"、"cloudy"、"missing"等 | 质量标识字段名、标识值含义、过滤阈值 |
| 地理定位 | 从文件读取经纬度坐标数组（可能是 1D scan/pixel 索引或 2D 数组）；验证坐标范围合理性 | lat/lon 变量名、坐标有效范围、填充值 |
| 投影转换 | 若数据使用投影坐标（如极地立体、兰勃特），需投影参数（proj4 字符串或 CRS 对象）转换到目标投影或经纬度 | 源投影参数、目标投影、转换库（pyproj、rasterio） |
| 重采样到规则网格 | 定义目标网格（分辨率、范围）；使用最近邻（分类）、双线性（连续）或保守（通量）插值；处理扫描重叠区域 | 目标网格定义、插值方法、重叠像素处理策略 |
| 轨道拼接 | 按时间排序多个轨道文件；在重叠区域取质量高或时间近的像素，或加权平均；记录拼接缝位置 | 轨道文件列表、重叠区选择规则、拼接缝处理 |
| 观测几何变量 | 保留或使用卫星天顶角、方位角、太阳天顶角等辅助变量；高角度观测可设置掩膜阈值 | 角度变量名、掩膜阈值（如 >70° 剔除） |
| 时间标注 | 卫星扫描线或像素可能有独立时间戳；统一到文件时间或保留像素级时间用于精确匹配 | 时间变量维度、时间精度要求 |

## 六、垂直层处理

| 垂直坐标 | 处理要点 |
|---|---|
| 压力层 | 常用于大尺度环流；插值时可在 pressure 或 log-pressure 坐标上处理 |
| 高度层 | 适合近地层或边界层分析；需明确相对地面高度还是海拔高度 |
| 模式层 | 依赖模式定义和地形；转换到压力层或高度层需要辅助变量 |
| 地表/单层 | 不可与压力层变量直接当作同一垂直坐标合并 |

常用垂直处理动作：

- 选择目标层位，如 850 hPa、500 hPa、200 hPa 或 2m、10m。
- 对多层变量统一层序和层单位，避免 pressure 从高到低或从低到高造成剖面倒置。
- 构建垂直剖面时，先确定剖面路径、插值方式和垂直坐标。
- 计算层结、稳定度、切变、水汽输送等派生量时，确保源变量同层、同时间、同网格。
- 地下压力层或被地形遮挡的层位应使用掩膜，不能直接参与近地统计。

### 垂直插值详细方法

| 插值类型 | 适用场景 | 具体方法 |
|---|---|---|
| 压力层线性插值 | 温度、湿度等状态量插值到目标压力层 | `value(p) = value(p1) + (value(p2)-value(p1)) * (p-p1)/(p2-p1)`，其中 p1 < p < p2 |
| 对数压力插值 | 位势高度等变量 | 在 log(p) 坐标插值：`value(p) = value(p1) + (value(p2)-value(p1)) * (log(p)-log(p1))/(log(p2)-log(p1))` |
| 高度层插值 | 近地层风速、温度廓线 | 在高度坐标线性插值或使用幂律廓线（如风速：`u(z) = u(z_ref) * (z/z_ref)^α`） |
| 模式层转压力层 | 数值模式输出转标准层 | 需读取模式层定义系数（如 sigma、hybrid）和地表气压，计算每层实际压力后插值 |
| 地形遮挡处理 | 地表气压低于目标压力层 | 标记或掩膜地下层位：`mask = (surface_pressure < target_pressure)` |
| 垂直积分 | 可降水量、整层水汽通量 | `integral = Σ value[k] * Δp[k] / g`，其中 Δp 为层厚，g 为重力加速度 |

## 七、多源数据对齐与融合

多源气象数据常见差异包括时间粒度、空间分辨率、变量定义、观测误差和覆盖范围。

| 对齐对象 | 推荐处理 |
|---|---|
| 时间 | 对齐到 valid_time；设定最近邻、插值或聚合策略 |
| 空间 | 选择目标网格；记录重网格方法和分辨率变化 |
| 垂直层 | 统一到共同压力层、高度层或目标层集合 |
| 变量 | 建立变量映射表，记录单位、符号、累计窗口 |
| 掩膜 | 统一海陆、地形、缺测和有效区域 |
| 统计口径 | 统一日界线、季节定义、区域边界和样本筛选条件 |

融合时保留来源标识和处理历史。训练数据中如混合观测、再分析和模式产品，应明确哪个数据源作为输入、标签、辅助变量或验证基准。

### 多源融合详细方法

| 融合方法 | 适用场景 | 具体实现 |
|---|---|---|
| 简单替换 | 优先级明确的数据源 | 按优先级顺序：`data = data_high_priority where valid, else data_low_priority` |
| 加权平均 | 多源观测或再分析融合 | `data_fused = Σ(weight_i * data_i) / Σ(weight_i)`；权重可基于误差或数据源可靠性 |
| 最近邻选择 | 时间或空间最接近优先 | 计算时空距离：`data = data_source[argmin(distance)]` |
| 方差最小融合 | 集合预报或多模式融合 | `weight_i = 1/variance_i`；融合方差：`1/variance_fused = Σ(1/variance_i)` |
| 偏差订正后融合 | 存在系统性偏差的数据源 | 先计算偏差：`bias = mean(data_source - reference)`，订正后融合：`data_corrected = data_source - bias` |
| 分区域融合 | 不同源在不同区域可靠 | 按海陆、纬度带、地形分区使用不同数据源或权重 |
| 变量级融合 | 部分变量来自不同源 | 建立变量-数据源映射表；确保所有变量时空对齐后组合 |

## 八、缺测、异常与掩膜处理

此处只描述处理规划中的通用方法，不替代专门的数据质量检查。

| 问题 | 常用处理 |
|---|---|
| 缺测值 | 统一为 NaN 或掩膜；统计时记录有效样本数 |
| 空间缺口 | 小范围可插值；大范围应保留掩膜或剔除样本 |
| 时间缺口 | 可按时间插值、窗口剔除或样本掩膜；训练时记录缺测比例 |
| 累计量重置 | 差分后出现负值时标记窗口边界或数据重置 |
| 物理越界 | 温度、湿度、风速、降水等可做范围约束标记 |
| 海陆/地形不适用 | 对 SST、海冰、土壤湿度等变量使用适用区域掩膜 |

模型训练场景中，缺测处理需形成可复现策略：填充值、掩膜通道、样本剔除阈值和归一化统计分母都要写入数据契约。

### 缺测与异常处理详细方法

| 处理方法 | 适用场景 | 具体实现 |
|---|---|---|
| 缺测值统一 | 不同格式缺测标记 | 识别 NaN、-999、9.96921e+36 等；统一转换：`data[data == fill_value] = np.nan` |
| 空间插值填补 | 小范围孤立缺测 | 最近邻：取周围有效值；双线性：`f(x,y)` 基于四邻点；仅适用缺测率 <10% 区域 |
| 时间插值填补 | 短时缺测 | 线性插值：`value(t) = (v1*(t2-t) + v2*(t-t1))/(t2-t1)`；仅适用瞬时量和短缺口（<3 步） |
| 气候态填充 | 长时缺测或历史数据 | 使用同期多年平均：`value[missing] = climatology[day_of_year]` 或 `climatology[month]` |
| 样本剔除 | 大范围或关键时刻缺测 | 设置阈值：`valid_samples = samples[missing_ratio < threshold]`；threshold 通常 0.2-0.3 |
| 掩膜通道 | 训练时保留缺测位置信息 | 添加二值掩膜：`mask = 1 where valid, 0 where missing`；作为额外输入特征 |
| 异常值检测 | 物理不合理值 | 范围检查：`outlier = (T < -100) | (T > 60)`；3σ 准则：`outlier = abs(value - mean) > 3*std` |
| 异常值处理 | 检测到的异常 | 设为缺测：`data[outlier] = np.nan`；或截断：`data = np.clip(data, min_val, max_val)` |
| 海陆掩膜应用 | 海洋/陆地专属变量 | SST 仅保留海洋：`sst[land_mask] = np.nan`；土壤湿度仅保留陆地：`sm[ocean_mask] = np.nan` |
| 地形掩膜应用 | 地下压力层 | 掩膜地表气压低于目标层的格点：`data[surface_pressure < level_pressure] = np.nan` |

## 九、统计、聚合与分析前处理

通用统计动作可用于分析，也可用于生成训练标签、基准特征或数据摘要。

| 处理动作 | 适用对象 | 注意事项 |
|---|---|---|
| 描述统计 | 连续变量、区域样本、时间样本 | 同时输出有效样本数和缺测率 |
| 分组聚合 | 小时、日、月、季节、区域、层位 | 对累计量和状态量使用不同聚合函数 |
| 面积加权平均 | 规则经纬度网格区域平均 | 使用纬度权重或网格面积 |
| 分位数统计 | 极端阈值、归一化、稳健摘要 | 样本量不足时阈值不稳定 |
| 距平计算 | 气候态、异常场、模型输入特征 | 基准期和分组方式必须一致 |
| 平滑/滤波 | 时间序列、空间场 | 记录窗口长度，避免泄漏未来信息 |
| 重采样 | 不同时间粒度或空间粒度 | 先区分瞬时、平均、累计和通量 |
| 分箱 | 纬度带、海拔带、温度区间 | 记录边界闭合方式 |

## 十、模型训练数据集构建

当目标是训练、微调、评估或推理数据准备时，应把气象处理结果组织为可复现样本。

| 主题 | 推荐做法 |
|---|---|
| 样本索引 | 使用 time、valid_time、lead_time、level、lat/lon 或 station 建立唯一索引 |
| 输入窗口 | 明确历史窗口长度、步长、变量、层位和空间范围 |
| 预测标签 | 明确目标变量、预测时效、聚合窗口和标签生成规则 |
| 特征对齐 | 所有输入特征必须相对样本时刻可获得，避免未来信息 |
| 空间切片 | 记录 patch 大小、步幅、边界填充和投影信息 |
| 归一化 | 按训练集统计均值/标准差或分位数，验证/测试集复用训练统计 |
| 数据划分 | 优先按时间、空间或事件划分，避免相邻样本泄漏 |
| 样本权重 | 可按区域、季节、事件强度或类别不平衡设置 |
| 输出格式 | NetCDF/Zarr 适合多维数组；Parquet/CSV 适合表格样本；Tensor 格式需保留元数据 |

典型训练数据处理链：

1. 读取并统一变量、单位、时间、空间和垂直坐标。
2. 裁剪目标区域和时间范围，应用有效掩膜。
3. 派生必要变量，如风速、距平、累计量差分、垂直切变。
4. 对齐输入特征和标签时刻，构建样本索引。
5. 生成窗口、patch 或站点样本，并记录缺测策略。
6. 按训练/验证/测试划分计算归一化统计并导出数据契约。

## 十一、分析与可视化数据准备

当目标是分析、诊断或可视化时，应优先保留物理含义和坐标元数据。

| 目标数据形态 | 常用准备 |
|---|---|
| 时间序列 | 区域平均、站点提取、日/月聚合、滑动统计 |
| 空间场 | 目标时刻或聚合时段选择、重网格、掩膜、单位转换 |
| 剖面 | 选择站点/区域/路径，统一垂直坐标，构造 level-time 或 level-distance 数据 |
| 截面 | 沿经纬度、任意路径或等值面插值，记录路径定义 |
| 概率/集合摘要 | 计算均值、离散度、分位数、概率阈值 |
| 对比分析 | 保证两个数据源的时间、空间、层位、单位和统计口径一致 |

分析前处理不应提前丢弃关键坐标、属性和掩膜；输出应能追溯每个统计量的样本范围和处理方法。

## 十二、常见变量族

本节只提供通用变量族，不限定具体数据源。若使用 ERA5 等平台数据，应再读取对应元信息文件确认变量名、单位和层位。

| 变量族 | 常见变量 | 常用处理 |
|---|---|---|
| 近地面气象 | 2m 温度、2m 露点、10m 风、地表气压 | 单位转换、日统计、风速/风向派生、站点或区域提取 |
| 降水与水文 | 总降水、降雪、蒸发、径流、土壤湿度 | 累计量差分、窗口聚合、非负约束、流域或区域平均 |
| 辐射与能量 | 短波、长波、净辐射、感热、潜热 | 通量方向统一、累计/平均口径统一 |
| 压力层温度 | temperature at pressure levels | 垂直插值、层结、距平、剖面 |
| 压力层风 | u/v wind at pressure levels | 风速、切变、急流、水汽输送 |
| 位势高度 | geopotential/geopotential height | 单位转换、环流场、梯度相关派生量 |
| 湿度 | 相对湿度、比湿、可降水量 | 湿度转换、水汽通量、缺测和物理范围约束 |
| 海洋表面 | SST、海冰、海面风、浪高 | 海陆掩膜、海冰边界、海气变量对齐 |
| 陆面 | 土壤温湿度、积雪、植被、地表温度 | 地形/土地类型掩膜、层深或类别处理 |

## 十三、质量检查与验证

数据处理完成后应进行质量检查，确保物理含义正确和数据可用性。

| 检查项 | 检查方法 | 处理建议 |
|---|---|---|
| 元数据完整性 | 检查变量单位、坐标、时间含义、累计窗口和层位定义 | 缺少关键元数据时应从数据源文档补充，不应猜测 |
| 时间语义一致性 | 区分 reference_time、lead_time、valid_time，验证时间对齐逻辑 | 预报数据需明确标注时间类型，避免有效时刻错位 |
| 数值范围合理性 | 检查温度、湿度、风速、降水等是否在物理可能范围内 | 设置合理阈值标记异常值，记录异常比例 |
| 累计量处理正确性 | 验证累计量差分后无大量负值，聚合方式符合物理含义 | 差分出现负值需检查重置点；累计量不可做时间插值 |
| 空间坐标一致性 | 验证多源数据的经纬度、投影参数和网格对齐 | 形状相同不代表坐标一致，需比对坐标数组值 |
| 面积权重应用 | 区域统计时检查是否使用纬度权重或网格面积 | 规则经纬度网格平均应使用 `cos(latitude)` 权重 |
| 插值方法适用性 | 验证连续变量、分类变量、守恒量使用了正确插值方法 | 降水等守恒量用保守插值；分类变量用最近邻 |
| 地形一致性 | 检查近地变量处理时源数据和目标网格的地形差异 | 跨分辨率处理地表气压、温度和降水时需记录地形影响 |
| 缺测率统计 | 计算时间、空间、垂直维度的有效样本比例 | 高缺测率区域或时段应在后续分析中标记或剔除 |
| 样本独立性 | 验证训练/验证/测试集划分避免了时空临近样本泄漏 | 优先按时间连续段划分，避免打乱后随机抽样 |
| 归一化一致性 | 验证验证集、测试集使用了训练集统计量 | 所有数据集应使用同一归一化参数，不可各自计算 |
| 数据源标识 | 检查融合数据是否保留了来源标识和处理历史 | 多源数据需记录每个变量或样本的数据源类型 |
