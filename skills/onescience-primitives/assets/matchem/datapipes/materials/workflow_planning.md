# description

该规划知识用于指导 agent 在材料机器学习势任务中把原始原子结构数据组织成可训练数据流。决策重点是根据任务目标和数据形态选择合适的数据读取方式、字段变换、采样策略和批次构造方式，并在进入模型训练前验证 AtomicData 字段是否完整、物理语义是否一致、统计量是否同源。

# when_to_use

- 需要为 UMA 或类似 universal MLIP 模型准备材料原子体系数据。
- 数据可以表示为 ASE 原子结构，或可以转换成 ASE 数据库、ASE LMDB、结构文件目录或多帧结构文件。
- 任务包含能量、力、应力、分子电荷/自旋、晶体周期边界、表面固定原子或多数据域训练。
- 需要按多个数据集、多个 split 或多个采样比例组织训练数据。
- 需要把不同数据源规范成统一 AtomicData 字段协议。

不适合优先使用：

- 数据不是原子结构。
- 任务是 MACE extxyz 训练主线且已有专用 extxyz 工具链。
- 已经拥有完全合格的 batched AtomicData，只需要直接推理。
- 主要问题是标签质量或单位不明，而不是数据 pipeline 构建。

# inputs

- 任务类型：energy-only、EF、EFS、分子任务、晶体任务、表面/吸附任务、多数据集混合任务。
- 原始数据格式：数据库、结构文件、多帧结构文件、已转换的 ASE 数据。
- 数据路径：每个 split 的路径、目录、glob 或列表。
- 标签字段：energy、forces、stress、charge、spin、tags、fixed、sid 等字段的来源。
- 数据统计：元素参考能量、归一化常数、元素覆盖、原子数元数据。
- 采样需求：是否限制最大原子数、是否抽样、是否平衡多数据集、是否做温度采样。
- 运行需求：batch size、num workers、batch sampler、collate 行为、是否分布式训练。

# outputs

- 数据处理方案：选择数据库读取、结构文件读取或多帧结构读取。
- 字段协议：定义单样本 AtomicData 和 batch AtomicData 必须包含的字段。
- 变换策略：说明需要补哪些字段、哪些字段必须真实存在、哪些字段需要 reshape 或映射。
- 采样策略：定义 split 内部筛选和多数据集混合策略。
- 启动方式：给出配置式训练入口或 Python API 调用方式。
- 验证清单：字段完整性、shape、dtype、PBC、charge/spin、stress、统计量和 batch 拼接结果。
- 风险处理：针对格式错误、字段缺失、metadata 缺失、采样比例缺失和 loss 异常给出兜底。

# procedure

1. 确认任务是否属于材料原子势数据准备；若是 MACE extxyz 主线，转向 MACE 专用数据处理。
2. 检查原始数据是否能转换为 ASE 原子结构；不能转换时，先做数据格式转换。
3. 为每个数据域定义数据名称、数据格式和 split 路径。
4. 明确任务标签：energy-only 只需能量，EF 需要能量和力，EFS 还需要应力。
5. 明确特殊字段：分子任务检查 charge/spin，表面和吸附体系检查 fixed/tags，晶体任务检查 cell/PBC。
6. 配置字段读取策略，保证监督标签进入 AtomicData。
7. 配置数据变换策略，将样本整理为统一字段协议。
8. 若存在字段命名差异，配置字段映射，避免在模型层临时兼容。
9. 配置采样策略：先用单数据集显式比例跑通，再扩展到多数据集均衡或温度采样。
10. 配置 batch sampler 和 collate，确保变长原子结构能正确拼接。
11. 做小批量检查：读取样本、执行变换、拼接 batch、检查字段和 shape。
12. 再进入完整训练，并监控 loss 尺度、分数据集误差和异常样本。

# constraints

- 数据格式声明必须与真实数据一致。
- 任务统计量必须与训练数据同源。
- 电荷、自旋、应力、固定原子和周期边界不能在物理语义不明时随意补默认值。
- 元数据过滤依赖可用元数据。
- 多数据集采样比例必须覆盖所有参与训练的数据域。
- 不同 DFT 设置、单位或标签定义的数据集混合前必须显式说明并设计对应任务策略。
- 批次拼接后必须保留每个原子的结构归属，否则模型无法正确聚合结构级量。

# next_phase_recommendation

- 为目标数据集补充数据卡，记录格式、字段名、单位、PBC、charge/spin、stress 约定、元素覆盖和统计量来源。
- 先构建最小 train/val 数据流，完成单 batch 字段检查后再扩大数据量。
- 对 EFS 任务单独验证 stress shape、单位和 loss 归一化。
- 对 OMOL 或其他分子任务单独验证 charge/spin 和大真空盒策略。
- 对多数据集训练输出分数据域指标，避免总体指标掩盖某个数据域失败。
- 对大规模训练准备原子数元数据，以支持 max atoms 过滤和更稳定的 batch 规划。

# fallback

- 数据无法读取：先用 ASE 单独读取一个样本，确认格式和路径，再接入 pipeline。
- 数据格式声明错误：切换为匹配真实数据的读取方式，或先转换数据格式。
- 标签字段缺失：回到数据准备阶段补齐字段或设置正确字段映射。
- charge/spin 缺失：分子任务停止训练，补真实电荷/自旋后再继续。
- stress 异常：先统一 shape、单位和符号，再启用 EFS 训练。
- metadata 缺失：关闭依赖 metadata 的过滤，或先生成元数据。
- 多数据集采样失败：补齐每个数据域的采样比例，先退回单数据集 baseline。
- batch 拼接失败：先检查单样本字段，再检查不等长字段和 collate 规则。
