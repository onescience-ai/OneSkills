# Domain Profiles

本文件用于给 `onescience-primitives` 提供领域友好的语义模板，并在调用方未显式传入 `domain` 或现有上下文不足以稳定判断领域时，作为
**补充判定依据**。

## 领域模板使用原则

1. 领域模板只负责补充语义，不复制执行逻辑。
2. 领域模板用于帮助理解用户说法、识别约束和完善交接摘要。
3. 仅支持 `climate`、`cfd`、`matchem`、`bio`、`unknown` 五类输出，不再使用历史别名或扩展领域名。
4. 调用方已明确提供 `domain` 时优先使用调用方信息；只有在缺失或不可靠时，才依据本文件做回退判断。
5. 无法准确判断领域时，使用 `unknown`，不要强行误判。

## 标准化判定顺序

1. 先看调用方是否显式给出 `domain`。
2. 若未给出，再根据用户请求中的领域关键词、任务对象、数据类型和常见模型名称进行判断。
3. 若多个领域信号冲突，或只有很弱的泛化科研信号，则归为 `unknown`。
4. 输出值必须是以下之一：`climate | cfd | matchem | bio | unknown`。

## `climate`

常见信号：

- 气象数据
- 区域气象
- 天气预报
- 短期预报
- 中期预报
- 全球预报
- ERA5
- 再分析资料
- 气候预测
- 降水、温度、风场、气压、湿度、辐射
- 台风、对流、极端天气、数值天气预报

领域归类说明：

- 当任务明确围绕天气、气象、气候、再分析资料、预报场或大气变量时，归入 `climate`
- 当请求提到天气模型、气象数据管线、预报评估或气候场建模时，归入 `climate`
- 若只提到通用时空网格数据，但没有明确气象语义，不要直接判为 `climate`

## `cfd`

常见信号：

- CFD
- 流体、流场、空气动力学、翼型、绕流、涡脱落
- RANS、LES、Navier-Stokes、Euler 方程
- PDE、PDE 建模、PDE 代理模型、神经算子
- FNO、UNO、U-Net、Transolver、MeshGraphNet、PINO、DeepONet
- VTK、网格、非结构网格、点云、表面压力、压力系数、速度场、阻力/升力

领域归类说明：

- 当用户说的是 PDE 模型、PDE 数据集、PDEBench、神经算子或物理场代理，但没有明确指向气象、材料化学或生信时，默认归入 `cfd`
- 当任务明确是流体、翼型、管道、腔体、圆柱绕流、气动外形、流场预测或稳态/非稳态流体代理时，归入 `cfd`
- 当任务只是抽象数学求解、有限元泛化或未知物理领域时，可先标记为 `cfd` 候选，但若缺乏稳定流体信号则应回退为 `unknown`

## `matchem`

常见信号：

- 材料模拟
- 材料化学
- 晶体结构
- 分子性质预测
- 第一性原理 / 计算化学
- MACE / UMA / 其它材料模型 / MLIP / U-MLIP
- extxyz / xyz / ASE / LAMMPS / DeepMD 数据
- OC20 / OMAT / OMOL
- 能量、力、应力、势函数、结构弛豫、MD

领域归类说明：

- 当任务围绕材料体系、分子体系、原子结构、势函数或计算化学数据时，归入 `matchem`
- 当请求提到材料基础模型、MLIP、结构弛豫、性质计算或分子/晶体数据接入时，归入 `matchem`
- 若只是通用图学习或通用分子机器学习讨论，但没有稳定材料化学任务对象，可回退为 `unknown`

## `bio`

常见信号：

- 蛋白
- 基因
- 生物序列
- 结构预测
- 生信
- 组学 / 单细胞 / 空间组学
- 变异检测 / 群体遗传
- 蛋白设计 / 药物设计
- 公共生物数据库 / 临床变异 / 实验室数据

领域归类说明：

- 当任务明确围绕蛋白、核酸、组学、结构生物学、变异分析或生物数据库时，归入 `bio`
- 当请求提到 OneScience 生信模型推理、模型改造、序列建模或生物分析链路时，归入 `bio`
- 对传统生信流程、数据库查询和实验室数据任务，也可判为 `bio`，但不要默认等同于 OneScience 模型开发

## `unknown`

适用场景：

- 无法稳定识别具体领域
- 多个领域信号冲突
- 需求更偏通用工程实现而非领域语义
- 只有弱关键词，无法支撑可靠路由
